# 仓库分析报告（awesome-green-QAR）

## 1. 仓库定位
该仓库是一个“民航飞行（civil flight）方向论文与资源清单”项目，目标是汇总：
- 综述（Surveys）
- 基准与数据集（Benchmarks and Datasets）
- 方法类论文（Methods）
- 其他资源（Resources）

从 README 结构来看，项目偏向于“Awesome List/知识导航”而非可执行代码库。

## 2. 当前内容现状

### 2.1 文档结构
仓库当前主要包含：
- `README.md`（主列表）
- `CONTRIBUTING.md`（贡献指南）
- `LICENSE`（MIT）
- `img/`（配图）

没有看到源码目录（如 `src/`、`app/`、`scripts/`）或依赖配置文件（如 `requirements.txt`、`package.json`、`pyproject.toml`）。

### 2.2 README 完整度
README 已具备目录结构，但仍以占位内容为主：
- 多处使用 `[Paper Title](paper_link)` 模板占位。
- “Methods”下面出现了一段与“LLM 参数高效微调/知识编辑”相关的 HTML 表格条目，主题与“civil flight”不一致，疑似误贴内容。
- 少量真实条目（如 hard landing safety assessment）已出现，说明仓库处于“从模板向真实清单迁移”的早期阶段。

### 2.3 贡献流程
`CONTRIBUTING.md` 规则简洁明确，强调三个最低标准：相关性、引用准确性、链接有效性。该规则适合轻量社区协作，但尚缺更细粒度的格式规范（例如 BibTeX 字段、标签体系、排序策略）。

## 3. 主要问题诊断
1. **主题一致性问题**：README 中存在明显偏题条目（LLM 方向），影响仓库专业性与可信度。
2. **信息密度不足**：大量占位符导致用户难以直接使用。
3. **缺少分类标签体系**：目前分类较粗，不利于按任务场景检索（例如预测、异常检测、风险评估、着陆阶段等）。
4. **缺少质量控制机制**：尚无自动检查（死链检查、重复条目检查、格式规范检查）。
5. **缺少更新透明度**：没有 changelog、最近更新分区或时间戳策略。

## 4. 改进建议（按优先级）

### P0（立即）
- 清理与民航飞行主题不相关条目（尤其是 LLM 偏题表格内容）。
- 将占位符替换为首批高质量真实文献（建议先补齐 20~30 条）。
- 统一条目格式，例如：
  `作者. 题目. 会议/期刊, 年份. [Paper] [Code] [Dataset] [Notes]`

### P1（短期）
- 在 Methods 下建立更可检索的二级标签：
  - Flight Phase（起飞/爬升/巡航/进近/着陆）
  - Task Type（状态估计、轨迹预测、异常检测、安全评估、维护诊断）
  - Data Source（QAR/FDR/ADS-B/ATC）
- 增加“数据集卡片”字段：规模、采样频率、公开性、许可协议、字段说明。

### P2（中期）
- 增加 CI 自动检查：
  - Markdown lint
  - URL 可用性检查（周期性）
  - 重复 DOI/标题检测
- 提供 `templates/new-paper.md`，降低贡献门槛并提升格式一致性。

## 5. 面向维护者的实践建议
- 采用“月度维护窗口”（例如每月 1 次）集中处理新增 PR 与死链修复。
- 在 README 增加“最近更新（Last updated: YYYY-MM-DD）”。
- 对高价值条目加一行“推荐理由（1 句话）”，提升列表可读性。
- 建立“纳入标准”与“排除标准”示例，减少低质量投稿。

## 6. 结论
该仓库方向正确、结构雏形已具备，适合作为 civil flight 研究导航入口；但当前仍处于模板化早期阶段。若优先完成主题清洗和首批核心文献补全，再配合轻量自动化质量检查，可较快提升为可用且可信的领域 Awesome List。
